‚úì
OS ‚Äî Processes, Threads, Scheduling, Deadlocks
Must know: Process vs Thread (memory model), CPU scheduling (FCFS, SJF, Round Robin, Priority), Deadlock conditions (mutual exclusion, hold and wait, no preemption, circular wait), deadlock prevention/avoidance (Banker's algorithm). Must-answer questions: "What happens when you type a URL?" "What is a context switch?" "Difference between process and thread?"

OS
‚úì
OS ‚Äî Memory Management: Virtual Memory, Paging, Segmentation
Virtual memory lets processes use more memory than physically available. Paging: fixed-size frames, page tables, TLB. Segmentation: variable-size segments. Page replacement algorithms: FIFO, LRU, Optimal. Cache memory hierarchy: L1/L2/L3 ‚Äî understand why cache hits matter for performance. These concepts appear in system design interviews too.

DBMS ‚Äî SQL mastery: JOINs, Indexes, Transactions, ACID properties
SQL: INNER/LEFT/RIGHT/FULL JOIN, GROUP BY + HAVING, subqueries, window functions (ROW_NUMBER, RANK, LAG/LEAD). Indexes: B-tree vs Hash, when to index, index costs. ACID: Atomicity, Consistency, Isolation, Durability ‚Äî know each with an example. Practice: LeetCode SQL problems (top 30 most common).

DBMS ‚Äî Normalization, NoSQL, and database design
Normalization: 1NF, 2NF, 3NF, BCNF ‚Äî with examples. When to denormalize for performance. NoSQL types: Document (MongoDB), Key-Value (Redis), Column (Cassandra), Graph (Neo4j) ‚Äî when to choose each. CAP theorem: Consistency, Availability, Partition Tolerance ‚Äî can only have 2. This is system design foundation.

Computer Networks ‚Äî OSI model, TCP/IP, HTTP/HTTPS, DNS

HTTP methods: GET, POST, PUT, PATCH, DELETE ‚Äî semantics and idempotency. HTTPS: TLS handshake, certificates, symmetric vs asymmetric encryption. DNS resolution: browser ‚Üí OS cache ‚Üí resolver ‚Üí root nameserver ‚Üí TLD ‚Üí authoritative.
